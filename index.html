Industry

Healthcare

Healthcare is a prime cyber target: sensitive patient data, zero tolerance for downtime. Hospitals, clinics, insurers, vendors are all interconnected—one weak link risks the whole chain.

Posture

Defense (Protect)

Thesis Statement

The safest low-cost approach for healthcare email threats is an agentic screener that auto-quarantines suspected spam/phish (leveraging reasoning) into an IT review/training queue, minimizing user exposure and alert load without deleting or auto-blocking legitimate mail.

Key Constraints

"This isn't a complex hack problem—it's a human and budget problem."

BUDGET

Zero dollars for new security software

STAFF

Single IT Generalist (Director Li)

THREAT

Phishing → Human Error (Top Risk)

Design for the Protect Posture

Separation of Duties

One for risk assessment, one for action of quarantine or delivery

Guardrails

Stay in scope for specific duty. Strict whitelist for external tool usage

Fail Safe Default

Unsure? Quarantine for review

Zero Trust

Analyzes even internally sent mail (insider threat / compromised account)

User Reporting

Allows for user reporting if something falls through the cracks

Model Training

Allows for model training through human review in quarantine phase

Privacy & Safety (Healthcare Specific)

  • • No real PHI stored beyond what already exists in mail logs—store hashes/indicators, not full content in case of breach
  • • Redaction: mask MRNs/SSNs if parsed, redact names in examples fed back to training
  • • Least privilege: mail API scopes limited to read + move, no delete or external forward

Metrics (Defense Evaluation)

Effectiveness

Phishing catch-rate, spam reduction, user exposure reduction (% delivered of risky mail)

Quality

False positive rate (quarantined but legit), time-to-review/release

Fatigue

Alerts per analyst/day, queue depth, median explanations length

Learning

Precision/recall before vs after feedback updates

Safety

0 permanent deletions of mail • 100% reversibility

What AI Agents Look For in Emails

  • Sender checks: SPF/DKIM/DMARC failures, spoofed addresses
  • Headers: Strange sending paths, mismatched info
  • Links: Hidden or lookalike domains (h0spital.com)
  • Attachments: Macros, disguised files, password-protected zips
  • Message content: Urgent tone, requests for logins or money
  • Context: High-risk topics (EHR, billing, scheduling)

Role of AI Agents in Detection

Patient Health Information (PHI): Names, IDs, records removed before AI analysis

Feature extraction: AI pulls key signals (headers, links, text, files)

Reasoning step: Agent makes JSON "report card" (risk score, evidence, action)

Guardrails: Safety rules: read-only, schema checks, default "hold" if unsure

HITL review: Human-In-The-Loop: IT staff makes the final decision, nothing is deleted automatically
